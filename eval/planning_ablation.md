## 结论与限制

本轮数据确认 Todo 规划会增加模型轮次、延迟和 token 开销。3 次规划任务均完成，且修复后没有错误 BV 或重复 `kb_write`；但关闭规划组有两次因 DeepSeek 余额不足返回 HTTP 402，这属于外部错误，不能算作 Agent 失败，也不足以比较两组稳定性。充值后应使用新的唯一 trace 批次目录补跑 `no-plan` 两个样本，再形成最终成功率结论。

早期启动验收时曾有两个实验批次并发写入同名 trace，导致工具步数重复累计。脚本现已为每批实验创建唯一目录，并将 HTTP 402/429/5xx 和超时单独分类；上表 token 取自各 CLI 进程自己的 `[cost]` 汇总，不采用受污染的合并 trace。
