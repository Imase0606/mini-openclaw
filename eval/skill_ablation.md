# `video-summary` Skill 消融实验

## 实验设计

2026-07-13 使用 `deepseek-chat`、温度 0 和同一组确定性 Tool fixture，只改变是否注入 `video-summary` Skill。每组运行正常内容、内容不足、提示注入三个场景，共 3 次。

运行命令：

```bash
python -m eval.skill_ablation
```

## 结果

| 指标 | 有 Skill | 无 Skill |
| --- | ---: | ---: |
| 运行数 | 3 | 3 |
| 空内容识别率 | 100% | 100% |
| 无依据内容代理命中率 | 0% | 0% |
| 危险工具尝试 | 0 | 0 |
| 来源说明率 | 100% | 33.3% |
| 内容不足时诊断落盘率 | 100% | 0% |
| 平均延迟 | 31.5s | 21.0s |

## 结论

- 两组都没有产生检测到的无依据事实，也没有尝试危险工具，因此本实验**不能证明 Skill 进一步降低了编造率**。防编造的稳定底线主要来自 Tool 的内容充足度检查和权限层。
- Skill 的可证收益是稳定提供来源与可信度说明，并在内容不足时调用 `kb_write` 保存可审计诊断条目；无 Skill 组虽然诚实拒绝总结，但没有落盘诊断结果。
- Skill 增加了工作流步骤和 prompt 长度，本轮平均延迟增加约 10.5 秒。复杂视频任务可接受这一开销，简单查询不应加载该 Skill。

无依据内容指标采用固定禁止事实词表作为可重复代理，不等同于完整语义事实核查。原始输出保存在被忽略的 `.mini-openclaw/eval/skill_ablation_results.json`，演示时可结合 trace 人工复核。
