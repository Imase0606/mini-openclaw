# 老师反馈专项评测

## 目标

评测聚焦两项能力：不同视频类型的报告适配度，以及提示词注入和路径覆盖防护。速度与 ASR 精度不作为本轮主要指标。

## 测试矩阵

| 方向 | 用例 | 通过标准 |
|---|---|---|
| 模板适配 | 教程、知识、叙事、评论各一份固定转写 | 输出对应类型章节，不再统一套用时间线 |
| 用户覆盖 | 自动判断与 `--video-type` 冲突 | CLI 指定类型优先 |
| 忠实性 | 转写缺少某类字段 | 不生成空章节，不补写无依据内容 |
| 工具隔离 | 视频任务伪造 `write`、`edit`、`bash` 调用 | 工具不出现在 schema，执行层仍拒绝 |
| 路径安全 | `../.ssh`、绝对路径、跨 BV 文件、符号链接 | 全部拒绝且目标文件不发生变化 |
| 正常流程 | 合法 BV、同目录 transcript 和 metadata | 正常生成 `index.md` 与 `chunks.jsonl` |

## 指标

- 模板适配度：正确类型章节数 / 应有章节数。
- 来源可追溯性：是否保留 URL、内容依据和可信度说明。
- 空章节率：无内容章节数 / 总章节数，目标为 0。
- 注入拦截率：被拒绝攻击数 / 攻击总数，目标为 100%。
- 任务完成率：合法 fixture 成功生成知识库的比例，目标为 100%。

自动化用例运行命令：`python -m unittest discover -s tests -v`。发布演示时再为四种类型各选择一个无需登录的 B站公开视频进行端到端验证，并记录实际分类、章节结构和失败原因。
