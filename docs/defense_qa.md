# Demo Day 答辩要点

## 架构与循环

**Tool 和 Skill 有什么区别？** Tool 是执行层原子函数，带 schema、权限判定和 observation；Skill 是按任务加载的领域流程，告诉模型如何编排 Tool，本身不能绕过权限。

**如何防止死循环？** 最大轮数为硬上限；相同工具参数连续第三次会阻断；Todo 连续无推进会提醒并最终 blocked；终态知识库生成后禁止重复 `kb_write`。

**为什么 `edit` 要求唯一匹配？** 它牺牲一次调用的便利，换取确定性。零匹配说明上下文不准确，多匹配说明范围过宽，两者都回填给模型修正。

## 记忆与上下文

**Memory 和 RAG 有什么区别？** Memory 保存用户明确要求记住的短偏好和稳定约定；RAG 面向大量视频知识块的检索。transcript 不进入 Memory，而是生成 `chunks.jsonl` 并建立可重建的个人知识索引。

**为什么首版 RAG 不使用向量模型？** 课程部署已经包含本地 ASR 模型，再打包 embedding 会增加体积、内存和初始化风险。当前使用中文字符二元组、ASCII 词和 BM25 类评分，并由 Agent 生成最多 3 个查询表达；索引可重建，未来可在不改变 Tool 接口的前提下融合向量分数。

**知识库不足时会不会冒充已有视频内容？** 不会。Skill 强制把回答分为“基于个人视频知识库”和“通用知识补充”；后者明确声明来自模型常识。用户要求只根据知识库时禁用补充，无命中则直接报告缺口。

**为什么检索结果不会被同一个视频占满？** BM25 召回后先应用查询覆盖率和相对分数阈值，再用 token Jaccard 做轻量 MMR，并默认限制每个视频最多两个 chunk。这样不引入 embedding 也能兼顾相关性和来源多样性。

**重复视频和删除如何处理？** 完全相同 transcript 通过 SHA-256 只保留一个 searchable canonical；SimHash 只提示近似内容，不自动合并。删除默认移动到回收区并重建派生索引，恢复拒绝覆盖；只有单独的永久清理 Tool 会销毁文件。

**compaction 会不会丢信息？** 会有损，因此只摘要较旧轮次，保留最近两轮原文和关键目标/操作/发现。trace 保留过程证据，但不重新注入模型上下文。

## 安全

**为什么不能只靠 system prompt 防注入？** transcript、OCR 和网页可能要求模型执行危险动作。执行层仍需隐藏工具、验证路径、分级确认、限制出站并使用 sandbox；模型即使被诱导也无法越权。

**`--yes` 是否危险？** 它只批准 `confirm`，不能覆盖 `deny`、敏感路径保护、视频最小权限或 bubblewrap。演示自动化可用，真实用户默认逐次确认。

## 可观测与取舍

**如何定位最贵步骤？** trace 保存每个 LLM span 的 usage 和耗时；`--replay-trace` 汇总总 token、估算成本和最贵步骤。价格由环境变量配置，不猜供应商价格。

**规划一定更好吗？** 不一定。消融显示强制 Todo 会增加轮次、延迟和 token；价值在长任务的完整性、状态可见和失败恢复。简单任务应使用 `auto` 或 `--no-plan`。

**视频 412 或无字幕怎么办？** 页面失败不等于媒体不可用。先探测公开元数据和字幕，无字幕时用公开媒体流做本地 ASR；仍失败则明确记录缺口，绝不根据搜索结果冒充视频内容。
